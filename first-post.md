# Agentic Engineer: AI와 소프트웨어의 교차점

## 소개

AI Agent 기반 시스템을 개발하고 탐구합니다. AI와 소프트웨어의 교차점에서 에이전트가 어떻게 더 효과적으로 작동할지, 어떤 아키텍처 패턴이 유용한지 연구합니다.

## 이 블로그에서 다루는 내용

### 기술 리서치
- 새로운 AI 프레임워크와 라이브러리 탐구
- Agent 아키텍처 패턴 분석
- RAG (Retrieval-Augmented Generation) 활용법
- Multi-Agent 시스템 설계

### 개발 포스팅
- Next.js 16 + Supabase 스택 경험 공유
- 코드 스니펫과 라이브러리 리뷰
- 실무에서 마주친 문제와 해결책
- 튜토리얼 작성

### 프로젝트 포스트모템
- 완료한 프로젝트의 기술적 결정
- 아키텍처 선택 이유
- 배운 레슨과 개선점
- 실패에서 얻은 교훈

## 이 블로그의 기술 스택

이 블로그는 다음 기술 스택으로 구축되었습니다:

- **Frontend:** Next.js 16.1.6 + React 19.2.3
- **Styling:** TailwindCSS 4
- **Backend:** Supabase (PostgreSQL + Authentication + Realtime)
- **Deployment:** Vercel (GitHub → Vercel CI/CD)

## 앞으로의 계획

- 지속적인 콘텐츠 생산 (주 1-2회 포스팅)
- 개인 프로젝트 포트폴리오 확장
- 기술 블로그로서의 품질 향상
- 수익화 가능성 탐색
